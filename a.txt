3) Backend: new failure endpoints
I added two new Flask routes in backend/app.py without touching the existing observation/session search logic:

a) /api/failures/l0
Purpose: L0 Failed Observation Attributes:

Observation ID
RC ID
Dump Orbit
Dump Station
Error Message
Implementation:

Tries to use:
SCHEDULERINGESTNISAR (for L0 ingest/workorder + dumpingOrbit, sat_id, validFlag, ilevel)
PRODUCTMETAINFONISAR (for OBSERVATIONID, cridid, status_str)
Heuristic:
Rows with ilevel = '0' and validFlag != 'Y' are treated as L0 failures.
Returns JSON like:
{
  "data": [
    {
      "observation_id": "...",
      "rc_id": "...",
      "dump_orbit": 12345,
      "dump_station": "SAN",
      "error_message": "..."
    }
  ],
  "total": 1,
  "page": 1,
  "per_page": 50
}
If SCHEDULERINGESTNISAR / PRODUCTMETAINFONISAR are not present in the actual DB, the endpoint now gracefully returns:

{"data": [], "total": 0, "page": 1, "per_page": 50}
(no 500 error).

b) /api/failures/dpgs
Purpose: DPGS Failed Observation Attributes:

Observation ID
RC ID
Track
Frame
Work Order ID
Error Message
Implementation:

Uses the scene table.
Treats a record as “DPGS failed” if any of the many *_error_msg columns is non-empty (e.g. produceRIFG_error_msg, produceRUNW_error_msg, … including _Lerror_msg variants).
Returns JSON like:
{
  "data": [
    {
      "observation_id": "...",
      "rc_id": "...",
      "track": 123,
      "frame": 45,
      "work_order_id": "WO123",
      "error_message": "..."
    }
  ],
  "total": 1,
  "page": 1,
  "per_page": 50
}
If scene doesn’t exist in the current DB, it also falls back to { data: [], total: 0 } instead of 500.
